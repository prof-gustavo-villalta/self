**Objetivo:** Alinhar o plano de curso, estabelecer o fluxo de trabalho em
laboratório e realizar o setup inicial do ambiente Flutter (Normal e Portátil).

---

## 1. Abertura e Contrato Didático (50 minutos)

### Guia para o Professor:

- **Apresentação Profissional:** Explique sua experiência com desenvolvimento
  mobile. No noturno, os alunos valorizam muito a experiência de mercado.
- **Plano de Voo:** Apresente o projeto semestral (PjBL). Deixe claro que o app
  evoluirá a cada semana.
- **Combinados:** Defina regras de uso do laboratório, prazos de entrega e como
  as evidências (commits/prints) serão coletadas.
- **Pergunta para a Turma:** "Quem aqui já tentou criar um app ou tem uma ideia
  de problema que um app poderia resolver?".

---

## 2. Visão Geral: O Ecossistema Flutter (50 minutos)

### Guia para o Professor:

- **Teoria Rápida:** Use o
  [Material 02](/self/courses/2026-1-DS-Prog-Aplicativos-Mobile-II-Novotec-Noturno/materiais/material-02)
  (Slides). Foque em: "Por que Flutter?", "Multiplataforma" e "Tudo é Widget".
- **Demonstração:** Mostre um app funcional rodando no celular e o código no VS
  Code. O impacto visual ajuda a motivar para a fase chata do setup.
- **Pergunta para a Turma:** "Qual a diferença entre um app nativo e um
  multiplataforma?".

---

## 3. Estratégia de Setup: Normal vs. Portátil (50 minutos)

### Guia para o Professor:

- **Cenário de Lab:** Se os computadores da ETEC não permitem instalação no C:,
  direcione para o
  [Material 03](/self/courses/2026-1-DS-Prog-Aplicativos-Mobile-II-Novotec-Noturno/materiais/material-03)
  (Setup Portátil).
- **Cenário de Casa:** Recomende a instalação padrão do
  [Material 01](/self/courses/2026-1-DS-Prog-Aplicativos-Mobile-II-Novotec-Noturno/materiais/material-01).
- **Ação:** Inicie o download do Flutter SDK e do VS Code simultaneamente para
  ganhar tempo.

---

## 4. Oficina: "Flutter Doctor" e Triagem (50 minutos)

### Guia para o Professor:

- **Mão na Massa:** Os alunos devem rodar o `flutter doctor` no terminal.
- **Triagem:** Ajude os alunos a lerem o "semáforo" do Flutter Doctor. Erros de
  Android Toolchain são comuns aqui.
- **Dica:** Se o emulador for pesado, priorize o uso de celulares físicos com
  cabo USB.

---

## 5. Fechamento e Registro de Progresso (50 minutos)

### Guia para o Professor:

- **Consolidação:** Garanta que todos criaram a pasta base do curso no Google
  Drive ou Github.
- **Checklist:** Verifique quem conseguiu o "Hello World" (projeto inicial do
  contador).
- **Pergunta para a Turma:** "Qual foi a maior dificuldade no setup hoje?".

## 7. Dicas de Ouro para o Professor

1. **Cabo USB é Vida:** Peça para os alunos trazerem sempre seus cabos USB
   originais. Cabos de má qualidade são a maior causa de falhas de conexão com o
   VS Code.
2. **Ambiente Portátil:** Tenha sempre 3 ou 4 pendrives com o Flutter SDK e o VS
   Code "zipados" para entregar aos alunos que tiverem problemas com a rede da
   escola.
3. **Não trave no Erro:** Se um aluno não conseguir resolver o setup em 20 min,
   peça para ele fazer dupla com quem conseguiu para não perder a explicação
   pedagógica.
