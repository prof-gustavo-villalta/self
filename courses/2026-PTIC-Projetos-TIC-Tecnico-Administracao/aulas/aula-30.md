# Aula 30 - Consolidação Lógica

**Data:** 29/06/2026  
**Duração:** 1 hora-aula (50 min) **Local:** Laboratório de Informática  
**Objetivo:** Revisar e integrar os conceitos de pensamento computacional
(decomposição, padrões, abstração e algoritmos) através da resolução de um
desafio complexo.

---

## 1. Abertura: A Jornada da Lógica (10 minutos)

### Conectando os Pontos

Aprendemos a dividir problemas, a encontrar repetições e a tomar decisões. Hoje
é o dia de juntar tudo isso. Um profissional de administração não precisa ser um
programador, mas precisa pensar como um para organizar processos eficientes.

### Guia para o Professor

- **Instruções Didáticas:** Faça uma breve retrospectiva das ferramentas usadas:
  Draw.io, PSeInt, Scratch. Mostre que o foco nunca foi a ferramenta, mas sim o
  **raciocínio**.
- **Perguntas para a Turma:**
  - "Qual foi o conceito mais difícil de entender até agora?"
  - "Como o pensamento computacional pode ajudar a organizar sua vida pessoal?"
- **Dicas de Condução:** Use o quadro para criar um "mapa mental" da lógica.

---

## 2. Teoria: Refatoração e Otimização (15 minutos)

### Melhorando o Caminho

Um algoritmo que funciona pode ser melhorado.

- **Clareza:** Nomes de variáveis fáceis de entender (ex: `precoTotal` em vez de
  `p`).
- **Eficiência:** Evitar repetições desnecessárias.
- **Validação:** Garantir que o sistema não quebre se o usuário digitar algo
  errado.

### Guia para o Professor

- **Instruções Didáticas:** Mostre dois códigos que fazem a mesma coisa: um
  confuso e outro limpo. Pergunte qual eles prefeririam dar manutenção daqui a
  um ano.
- **Perguntas para a Turma:**
  - "O que acontece se eu digitar uma letra onde o programa espera um número?"
  - "Por que documentar (comentar) o código é importante?"
- **Dicas de Condução:** Introduza o conceito de "Código Limpo" (Clean Code) de
  forma bem básica.

---

## 3. Atividade Prática

Os alunos devem desenhar um fluxograma e depois criar o código para:

1. Perguntar a distância de uma viagem (em km).
2. Perguntar o tipo de combustível (1-Gasolina, 2-Etanol).
3. **Lógica de Consumo:**
   - Gasolina faz 12km/l | Etanol faz 9km/l.
4. Calcular quantos litros serão necessários.
5. Se a distância for > 100km, o programa deve sugerir uma parada para descanso
   (usando `Se`).
6. Se o combustível for Etanol e a distância > 200km, avisar sobre a autonomia.

### Guia para o Professor

- **Instruções Didáticas:** Este é um exercício de integração. Deixe-os baterem
  a cabeça um pouco antes de dar a resposta. Circule validando a lógica
  matemática.
- **Perguntas para a Turma:**
  - "Como vocês calcularam a quantidade de litros?" (Divisão: distância /
    consumo).
  - "Dá para fazer esse programa usando apenas o `Se` ou precisa de repetição?"
    (Neste caso, apenas `Se`).
- **Dicas de Condução:** Valorize quem usar mensagens de alerta claras para o
  motorista.

## 5. Dicas de Ouro para o Professor

1. **Trabalho em Dupla:** Este desafio funciona muito bem em duplas, onde um
   pensa na lógica e o outro digita (Pair Programming).
2. **Papel e Caneta:** Incentive-os a desenhar o fluxo no papel antes de ir para
   o computador.
3. **Personalização:** Peça para adicionarem o custo da viagem (pedindo o preço
   do combustível).
