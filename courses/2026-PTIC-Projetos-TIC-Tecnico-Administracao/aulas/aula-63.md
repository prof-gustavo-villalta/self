# Aula 63 - Projeto: Documentação Final

**Data:** 10/11/2026  
**Duração:** 1 hora-aula (50 min) **Local:** Laboratório de Informática  
**Objetivo:** Produzir o manual técnico e o relatório final do projeto,
registrando as ferramentas utilizadas, os prompts de IA e as instruções de uso
para o usuário final.

---

## 1. Abertura: O Registro do Sucesso (11 minutos)

### Por que Documentar?

Um projeto brilhante que ninguém sabe usar ou que ninguém sabe como foi feito
não tem valor profissional. Hoje vamos registrar a inteligência por trás do
nosso trabalho: as ferramentas que escolhemos, os prompts que criamos e o manual
para o usuário que vai herdar essa solução.

### Guia para o Professor

- **Instruções Didáticas:** Peça para os grupos abrirem todos os arquivos
  gerados até agora. Mostre exemplos de manuais de usuário simples (ex: manuais
  de eletrodomésticos ou software).
- **Perguntas para a Turma:**
  - "O seu projeto funcionaria se você não estivesse lá para explicar?"
  - "Quais os principais cuidados que o usuário deve ter com a sua ferramenta?"
- **Dicas de Condução:** Foque na **Clareza de Comunicação**.

---

## 2. Teoria: Relatório Técnico e Manual (17 minutos)

### O que deve conter?

1. **Introdução:** Qual o problema e qual a solução proposta.
2. **Ferramentas:** O que foi usado (Word, Excel, ChatGPT, etc.).
3. **Registro de IA:** Quais os prompts principais e onde a IA foi aplicada.
4. **Instruções de Uso:** Passo a passo para o usuário operar a ferramenta.
5. **Conclusão:** O que o grupo aprendeu com o projeto.

### Guia para o Professor

- **Instruções Didáticas:** Ensine a usar os recursos de **Sumário Automático**
  e **Estilos de Título** no Word, que aprendemos no início do ano.
- **Perguntas para a Turma:**
  - "Quem é o responsável por formatar esse relatório final nas normas
    (ABNT/Etec)?"
- **Dicas de Condução:** Ajude na organização visual do documento. Use capturas
  de tela (Screenshots) para ilustrar o manual.

---

## 3. Atividade Prática

Os grupos devem finalizar o documento de documentação:

1. **Redação:** Escrever as seções do relatório.
2. **Screenshots:** Capturar telas da solução funcionando para ilustrar o
   manual.
3. **Revisão:** Use a IA para revisar a gramática e a clareza do manual técnico.

**Tarefa:** Converta o relatório final em PDF e envie o link no formulário.

### Guia para o Professor

- **Instruções Didáticas:** Atue como o "Revisor Final". Verifique se o
  relatório está completo e profissional. Não esqueça da Capa e Contra-capa.
- **Perguntas para a Turma:**
  - "O que você achou mais difícil na documentação?"
- **Dicas de Condução:** Garanta que todos os nomes dos integrantes e as fontes
  de pesquisa estejam citados corretamente.

## 5. Dicas de Ouro para o Professor

1. **Atenção às Normas:** Revise se o sumário está atualizado e se as páginas
   estão numeradas.
2. **Backup Final:** Certifique-se de que nada se perdeu na última hora. O PDF é
   o "contrato" de entrega.
3. **Expectativa:** Avise que na próxima aula começaremos a preparação para a
   apresentação oral (Pitch).
