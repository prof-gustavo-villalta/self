# Aula 60 - Projeto: Prototipagem [Sábado Letivo]

**Data:** 31/10/2026  
**Duração:** 1 hora-aula (50 min) **Local:** Laboratório de Informática  
**Objetivo:** Realizar os primeiros testes funcionais da solução digital,
identificando erros de lógica, falhas na interface e alucinações da IA antes da
fase de refinamento.

---

## 1. Abertura: O Teste de Estresse (5 minutos)

### Será que funciona?

Hoje temos um tempo maior para testar nossa solução "de ponta a ponta". O
protótipo é o projeto incompleto, mas já capaz de mostrar como será o resultado
final. O objetivo não é ser perfeito, mas é descobrir onde ele falha para
podermos corrigir a tempo.

### Guia para o Professor

- **Instruções Didáticas:** Explique o conceito de **Teste de Usuário**. Peça
  que cada grupo eleja um "Testador Externo" (aluno de outro grupo) para usar
  sua ferramenta sem ajuda.
- **Perguntas para a Turma:**
  - "O que acontece se eu digitar uma data errada no seu formulário?"
  - "A solução é intuitiva?"
- **Dicas de Condução:** Aproveite o tempo extra do sábado para fazer
  orientações individuais mais profundas.

---

## 2. Teoria: Tipos de Testes (8 minutos)

### O que testar hoje?

1. **Teste de Lógica:** As fórmulas e fluxogramas estão batendo?
2. **Teste de Interface:** Os botões funcionam? O texto está legível?
3. **Teste de Conteúdo IA:** As respostas da IA são úteis e corretas?
4. **Teste de Integração:** O dado sai da planilha e vai para o relatório
   corretamente?

### Guia para o Professor

- **Instruções Didáticas:** Ensine a técnica do **"Testador Ingênuo"** — alguém
  que não conhece o projeto e tenta usá-lo seguindo apenas as instruções
  básicas.
- **Perguntas para a Turma:**
  - "O seu usuário vai precisar de um manual ou ele consegue usar sozinho?"
- **Dicas de Condução:** Use o sábado para reforçar o clima de cooperação entre
  os grupos.

---

## 3. Atividade Prática

Os grupos devem realizar o Ciclo de Prototipagem:

1. **Executar:** Use a solução do início ao fim com dados reais.
2. **Documentar:** Anote cada erro ou dificuldade que aparecer.
3. **Corrigir:** Tente resolver os erros mais críticos imediatamente.
4. **Feedback:** Convide alguém de outro grupo para testar por 5 minutos.

**Tarefa:** Crie um arquivo "Log_de_Erros.docx" na pasta do grupo com pelo menos
3 ajustes que foram feitos hoje.

### Guia para o Professor

- **Instruções Didáticas:** Estimule a autocrítica. Se algo não está
  funcionando, é melhor mudar o caminho agora do que no dia da apresentação.
- **Perguntas para a Turma:**
  - "Qual foi o erro mais engraçado ou inesperado que surgiu no teste?"
- **Dicas de Condução:** Gerencie o cansaço dos alunos (faça pausas) para manter
  a produtividade alta.

## 5. Dicas de Ouro para o Professor

1. **Olhar Crítico:** O professor deve atuar como o "Cliente Chato" que faz
   perguntas difíceis.
2. **Foco no Essencial:** Se o grupo está perdendo tempo com detalhes de design
   e a lógica não funciona, mande-os voltar para a lógica.
3. **Celebração:** O sábado letivo é cansativo. Termine a aula mostrando os
   pontos positivos de cada protótipo.
