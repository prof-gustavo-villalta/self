# Aula 61 - Projeto: Refinamento

**Data:** 03/11/2026  
**Duração:** 1 hora-aula (50 min) **Local:** Laboratório de Informática  
**Objetivo:** Refinar a solução digital com base nos feedbacks dos testes da
prototipagem, focando em usabilidade e na melhoria da experiência do usuário
(UX).

---

## 1. Abertura: Feedback é Ouro (11 minutos)

### O que os testes nos disseram?

Na última aula, testamos a solução até ela falhar. Hoje vamos analisar esses
erros e, principalmente, os feedbacks recebidos. Se o "testador ingênuo" não
entendeu um botão, o erro é do design, não do usuário. Vamos polir o diamante!

### Guia para o Professor

- **Instruções Didáticas:** Peça para os grupos abrirem o arquivo "Log_de_Erros"
  criado no sábado letivo. Priorize as correções que impactam a facilidade de
  uso.
- **Perguntas para a Turma:**
  - "Qual foi o feedback mais surpreendente que vocês receberam?"
  - "A solução já está com a identidade visual que vocês queriam?"
- **Dicas de Condução:** Foque na **Experiência do Usuário (UX)**. Menos é mais!

---

## 2. Teoria: Usabilidade e Acessibilidade (17 minutos)

### Pilares do Refinamento

1. **Consistência:** Use as mesmas cores e fontes em todos os documentos e
   planilhas.
2. **Navegação:** É fácil voltar para o início? As instruções são claras?
3. **Acessibilidade:** Pense em contraste de cores e legibilidade de fontes.
4. **Alinhamento:** Tudo deve estar alinhado profissionalmente (nada de tabelas
   "soltas" no Word).

### Guia para o Professor

- **Instruções Didáticas:** Demonstre exemplos de "Bad UI" (interfaces ruins) e
  como corrigi-los. Mostre a importância de um "Guia de Estilo" simples para o
  projeto.
- **Perguntas para a Turma:**
  - "Se o seu usuário fosse daltônico, ele conseguiria usar a sua planilha?"
- **Dicas de Condução:** Incentive o olhar crítico sobre o próprio trabalho. O
  projeto deve ter aparência de "Produto Profissional".

---

## 3. Atividade Prática

Os grupos devem realizar os ajustes sugeridos:

1. **Polimento Visual:** Ajuste de cores, ícones e fontes.
2. **Textos de Ajuda:** Melhore as mensagens de erro e as instruções de uso.
3. **Fluidez:** Reduza o número de passos para o usuário chegar no resultado.

**Tarefa:** Informe no formulário qual a principal mudança de design/UX feita
hoje.

### Guia para o Professor

- **Instruções Didáticas:** Passe nos grupos perguntando: "Como eu sei o que
  fazer aqui?". Ajude na escolha de paletas de cores mais adequadas.
- **Perguntas para a Turma:**
  - "Onde o seu projeto mais se parece com uma solução real de mercado?"
- **Dicas de Condução:** Lembre-os que a apresentação oral está se aproximando.
  O visual ajuda muito no impacto da apresentação.

## 5. Dicas de Ouro para o Professor

1. **Atenção aos Detalhes:** O professor deve observar "rebarbas" como nomes de
   colunas errados, fórmulas expostas desnecessariamente ou cores que cansam a
   vista.
2. **Simplicidade:** Se o grupo está tentando colocar "efeitos demais", sugira a
   limpeza visual.
3. **Motivação:** Mostre como o projeto evoluiu desde o fluxograma até hoje.
