# Aula 22 - Lógica: Padrões e Abstração

**Data:** 02/06/2026  
**Duração:** 1 hora-aula (50 min) **Local:** Laboratório de Informática  
**Objetivo:** Identificar regularidades em dados administrativos através do
reconhecimento de padrões e utilizar a abstração para simplificar processos
complexos, focando apenas no que é essencial.

---

## 1. Abertura: Identificando o que se Repete (10 minutos)

### O Poder do Padrão

Reconhecer padrões é o que nos permite automatizar. Se um problema acontece
sempre do mesmo jeito, podemos criar uma solução padrão. Se uma planilha sempre
tem os mesmos campos, podemos criar um modelo (template).

### Guia para o Professor

- **Instruções Didáticas:** Mostre uma série de formulários de inscrição e peça
  para os alunos identificarem o que todos têm em comum (Ex: Nome, CPF,
  Telefone). Explique que esse é o "padrão".
- **Perguntas para a Turma:**
  - "O que se repete em todos os meses em uma empresa?"
  - "Por que é mais fácil preencher um formulário padrão do que escrever uma
    carta do zero?"
- **Dicas de Condução:** Enfatize que o reconhecimento de padrões poupa energia
  cerebral. Não precisamos reaprender a dirigir toda vez que entramos em um
  carro diferente.

---

## 2. Teoria: Abstração (Simplificando o Caos) (15 minutos)

### Filtrando o Ruído

- **Abstração:** Retirar detalhes desnecessários para focar no que importa.
- **Modelagem:** Criar uma representação simplificada da realidade (Ex: Um mapa
  não tem cada árvore, apenas o caminho).

### Guia para o Professor

- **Instruções Didáticas:** Demonstre a abstração com um "Mapa do Metrô". Ele
  não mostra as curvas exatas das ruas, mas apenas as estações e as cores das
  linhas. Isso é abstração aplicada ao design.
- **Perguntas para a Turma:**
  - "Ao cadastrar um cliente, o que é essencial saber? A cor favorita dele ou o
    CPF?"
  - "Por que o Google Maps simplifica a visão de satélite para a visão de ruas?"
- **Dicas de Condução:** Explique que o profissional de administração "abstrai"
  os problemas pessoais dos colaboradores para focar no desempenho profissional
  (dentro dos limites da ética).

---

## 3. Atividade Prática

Os alunos devem analisar 5 tipos de "problemas de clientes" em uma empresa
fictícia, identificar o padrão das reclamações e criar um formulário abstrato
(modelo) que contenha apenas as perguntas essenciais para resolver esses
problemas.

### Guia para o Professor

- **Instruções Didáticas:** Peça para criarem o formulário no Google Forms ou no
  Word. Eles devem justificar por que cada campo é essencial (abstração) e quais
  campos eles removeram por serem inúteis (ruído).
- **Perguntas para a Turma:**
  - "Quais dados se repetiram em todas as reclamações?"
  - "O que você deixou de fora do seu formulário? Por quê?"
- **Dicas de Condução:** Circule verificando se os alunos estão conseguindo
  diferenciar o que é "curiosidade" do que é "dado necessário".

## 5. Dicas de Ouro para o Professor

1. **A Abstração é Subjetiva:** O que é essencial para o financeiro pode não ser
   para o marketing. Ensine o aluno a olhar pela ótica do objetivo final.
2. **Padrão em IA:** Reforce que as IAs atuais nada mais são do que grandes
   motores de reconhecimento de padrões (tema do 2º semestre).
3. **Mantenha a Simplicidade:** Use a frase de Leonardo da Vinci: "A
   simplicidade é o último grau de sofisticação". Um modelo administrativo
   complexo demais ninguém usa.
