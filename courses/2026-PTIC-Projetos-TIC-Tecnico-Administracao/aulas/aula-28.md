# Aula 28 - Oficina: Scratch

**Data:** 22/06/2026  
**Duração:** 1 hora-aula (50 min) **Local:** Laboratório de Informática  
**Objetivo:** Aplicar os conceitos de lógica (sequência, decisão e repetição) em
uma ferramenta visual de programação em blocos.

---

## 1. Abertura: Programação sem Digitação (10 minutos)

### A Lógica Visual

Até agora, escrevemos códigos. Hoje, vamos "montar" códigos. O Scratch permite
focar na lógica sem se preocupar com o erro de um ponto e vírgula ou um
parêntese esquecido. É a mesma lógica usada para criar grandes sistemas, mas de
forma visual.

### Guia para o Professor

- **Instruções Didáticas:** Mostre a interface do Scratch (Ator, Palco, Blocos).
  Explique que cada bloco é uma instrução que já conhecemos (Se, Para, Escreva).
- **Perguntas para a Turma:**
  - "Vocês acham que dá para criar um sistema administrativo usando blocos?"
  - "Qual a vantagem de ver o código em cores e formatos diferentes?"
- **Dicas de Condução:** Faça uma demonstração rápida: arraste o bloco "Mova 10
  passos" e clique nele.

---

## 2. Teoria: Blocos e Eventos (15 minutos)

### Componentes do Scratch

1. **Eventos (Amarelo):** "Quando a bandeira verde for clicada" (O início de
   tudo).
2. **Controle (Laranja):** Onde estão o "Se", o "Repita" e o "Sempre".
3. **Sensores (Azul Claro):** Onde fazemos perguntas ao usuário (Pergunte e
   Espere).
4. **Variáveis (Laranja Escuro):** Onde guardamos dados (pontos, nomes, saldos).

### Guia para o Professor

- **Instruções Didáticas:** Mostre como os blocos se encaixam como LEGO.
  Explique que blocos arredondados são valores (variáveis) e blocos hexagonais
  são condições (verdadeiro/falso).
- **Perguntas para a Turma:**
  - "Onde fica o 'Leia' no Scratch?" (Resposta: Bloco Pergunte/Resposta).
  - "Como fazemos um loop infinito no Scratch?" (Resposta: Bloco 'Sempre').
- **Dicas de Condução:** Incentive os alunos a explorarem os sons e movimentos,
  mas foque na lógica dos dados.

---

## 3. Atividade Prática

Os alunos devem criar um pequeno projeto no Scratch onde um personagem:

1. Cumprimenta o usuário e pergunta o nome.
2. Pergunta qual setor ele procura (1-Secretaria, 2-Coordenação, 3-Biblioteca).
3. Usa um bloco `Se` para dar a resposta correta (ex: "A secretaria fica no
   bloco A").
4. Usa um loop `Repita` para fazer o personagem "dançar" ou comemorar após
   ajudar.

### Guia para o Professor

- **Instruções Didáticas:** Ajude-os a encontrar o bloco `Resposta` dentro de
  `Sensores`. É comum eles esquecerem que a resposta da pergunta fica guardada
  ali.
- **Perguntas para a Turma:**
  - "Como faríamos para o assistente perguntar novamente se a opção for
    inválida?"
  - "Podemos usar variáveis para contar quantos atendimentos o robô fez?"
- **Dicas de Condução:** Deixe-os escolherem atores e cenários criativos para
  aumentar o engajamento.

## 5. Dicas de Ouro para o Professor

1. **Conta Online:** Se a escola permitir, peça para criarem contas no site do
   Scratch para salvarem os projetos. Caso contrário, use a versão offline e
   salve o arquivo `.sb3`.
2. **Remix:** Mostre que eles podem ver o código de outros projetos para
   aprender.
3. **Erro Comum:** Alunos tentam encaixar blocos que não têm o formato correto.
   Explique que a forma geométrica do bloco indica o que ele faz.
