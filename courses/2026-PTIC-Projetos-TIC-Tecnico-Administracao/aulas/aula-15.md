# Aula 15 - Projeto Final: Definição e Escopo

**Data:** 15/06/2026  
**Duração:** 1 hora-aula (50 min)  
**Local:** Laboratório de Informática  
**Objetivo:** Definir um projeto final aberto em Flutter, preferencialmente
vinculado ao TCC dos alunos, com escopo viável para desenvolvimento e
apresentação.

---

## 1. Proposta do Projeto Final

O projeto final será um aplicativo desenvolvido em **Flutter**. O tema é aberto,
mas deve resolver um problema real, ter relação com a área técnica dos alunos e,
sempre que possível, apoiar o **TCC** do grupo.

Exemplos de caminhos possíveis:

- Aplicativo de apoio ao produto, serviço ou sistema do TCC.
- Protótipo funcional para demonstrar uma ideia do TCC.
- Tela administrativa, cadastro, consulta ou acompanhamento de dados.
- Aplicativo simples para usuário final, cliente, equipe interna ou comunidade.

O foco não é fazer um aplicativo grande. O foco é entregar uma versão pequena,
clara e funcional, com interface organizada e objetivo bem explicado.

---

## 2. Requisitos Mínimos

Cada grupo deve propor um aplicativo que tenha:

- Nome do projeto.
- Problema que o aplicativo tenta resolver.
- Público-alvo.
- Relação com o TCC, quando houver.
- Pelo menos 3 telas planejadas.
- Navegação entre telas.
- Uso de componentes Flutter como textos, botões, campos, listas, cards ou
  imagens.
- Repositório, pasta ou arquivo organizado para acompanhamento do professor.

Quando o grupo não tiver TCC definido, o projeto pode ser um app de tema livre,
desde que tenha utilidade clara e escopo possível.

---

## 3. Atividade da Aula

Em grupo, preencham o planejamento inicial do projeto:

1. Definam o nome do aplicativo.
2. Escrevam o problema que ele resolve em uma frase.
3. Descrevam quem usaria o aplicativo.
4. Listem as telas principais.
5. Escolham quais funcionalidades serão implementadas primeiro.
6. Criem ou organizem o projeto Flutter que será usado nas próximas aulas.

### Entrega Esperada

Ao final da aula, cada grupo deve apresentar ao professor:

- Tema do projeto.
- Relação com o TCC ou justificativa do tema livre.
- Lista inicial de telas.
- Funcionalidade principal que será demonstrada na entrega final.

---

## 4. Guia para o Professor

- Incentive grupos com TCC a transformar uma parte do TCC em aplicativo.
- Ajude a reduzir escopos grandes para uma versão demonstrável.
- Priorize projetos que os alunos consigam explicar e defender.
- Reforce que o Flutter deve ser usado para construir o aplicativo, não apenas
  para desenhar telas soltas.
