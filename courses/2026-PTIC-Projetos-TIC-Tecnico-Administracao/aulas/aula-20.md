# Aula 20 - Pensamento Computacional

**Data:** 30/05/2026 [Sábado Letivo]  
**Duração:** 1 hora-aula (50 min) **Local:** Laboratório de Informática  
**Objetivo:** Introduzir o conceito de pensamento computacional, explorando seus
quatro pilares (decomposição, reconhecimento de padrões, abstração e algoritmos)
como ferramentas para solução de problemas na gestão administrativa.

---

## 1. Abertura: Pensar como um Computador? (10 minutos)

### O Que é Pensamento Computacional?

Não se trata de aprender a programar, mas sim de aprender a **resolver
problemas** de forma estruturada, eficiente e repetível. É uma habilidade mental
que usamos antes mesmo de tocar em qualquer tecnologia.

### Guia para o Professor

- **Instruções Didáticas:** Lance um problema cotidiano: "Como você explicaria a
  um robô como fazer um café?". Peça para os alunos descreverem os passos.
  Mostre que qualquer falha na instrução (Ex: não dizer para ligar a tomada) faz
  o robô falhar.
- **Perguntas para a Turma:**
  - "Computadores são inteligentes ou apenas executam ordens muito bem?"
  - "Quais tarefas do seu dia a dia parecem automáticas, mas têm uma lógica por
    trás?"
- **Dicas de Condução:** Use a definição do PTD: "Solucionar problemas de forma
  crítica e criativa". Enfatize que o pensamento computacional é uma ferramenta
  de produtividade para o administrador.

---

## 2. Teoria: Os Quatro Pilares (15 minutos)

### A Estrutura da Solução

- **Decomposição:** Quebrar um problemão em probleminhas menores.
- **Reconhecimento de Padrões:** Identificar semelhanças e tendências (o que se
  repete?).
- **Abstração:** Focar no que é essencial e ignorar detalhes irrelevantes.
- **Algoritmos:** Criar o passo a passo lógico para a solução final.

### Guia para o Professor

- **Instruções Didáticas:** Demonstre os pilares com um caso prático: "Organizar
  a festa de formatura".
  - Decomposição: dividir em comida, local, música, convites.
  - Padrões: observar que todos os convites levam nome, data e local.
  - Abstração: não importa agora o sabor do bolo, mas sim o número total de
    pessoas.
  - Algoritmo: Definir o cronograma da festa minuto a minuto.
- **Perguntas para a Turma:**
  - "Por que é difícil resolver um problema sem antes decompor ele?"
  - "O que acontece se eu focar nos detalhes irrelevantes (abstração ruim) no
    início de um projeto?"
- **Dicas de Condução:** Use diagramas simples no quadro para ilustrar cada
  pilar.

---

## 3. Atividade Prática

Os alunos recebem o cenário de uma empresa com estoque atrasado, funcionários
insatisfeitos e clientes reclamando. Eles devem aplicar os 4 pilares para propor
uma estratégia de solução estruturada.

### Guia para o Professor

- **Instruções Didáticas:** Divida a turma em grupos. Eles devem entregar um
  pequeno roteiro (em papel ou digital) mostrando como aplicaram cada um dos 4
  pilares no caso proposto.
- **Perguntas para a Turma:**
  - "Qual foi o 'probleminha' mais crítico que vocês encontraram após decompor o
    caos?"
  - "Qual padrão de erro vocês identificaram nos dados da empresa?"
- **Dicas de Condução:** Circule entre os grupos e desafie-os a serem
  específicos. "Melhorar a empresa" não é um passo de algoritmo; "Treinar os
  funcionários no sistema X" é.

## 5. Dicas de Ouro para o Professor

1. **Evite o Tecnicismo:** O objetivo não é falar sobre bits e bytes, mas sobre
   raciocínio lógico. Use exemplos do mundo real (receita de bolo, manual de
   instrução de móvel) para facilitar.
2. **Valorize a Abstração:** Muitos alunos travam nos detalhes. Ensine-os que na
   fase de planejamento, simplificar é a maior virtude.
3. **Ponte com IA:** Explique que para dar bons prompts para uma Inteligência
   Artificial (tema do próximo semestre), precisamos usar muito o pensamento
   computacional.
