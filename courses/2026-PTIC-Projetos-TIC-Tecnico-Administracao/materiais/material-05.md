## Por que Acessibilidade é Importante?

- ♿ **Inclusão:** 24% da população brasileira tem alguma deficiência
- ⚖️ **Lei:** Lei Brasileira de Inclusão (LBI) exige acessibilidade
- 📈 **Mercado:** Produtos acessíveis atingem mais pessoas
- 💼 **Profissionalismo:** Demonstra preocupação com todos os usuários

---

## Checklist para Documentos

### ✅ Estrutura e Navegação

- [ ] Uso correto de **títulos hierárquicos** (Título 1, Título 2...)
- [ ] **Sumário** clicável (em documentos longos)
- [ ] **Numeração de páginas**
- [ ] **Cabeçalho e rodapé** consistentes

### ✅ Texto

- [ ] **Fonte sans-serif** (Arial, Calibri, Helvetica)
- [ ] **Tamanho mínimo 12pt** (preferencialmente 14pt)
- [ ] **Contraste adequado** (fundo claro, texto escuro)
- [ ] **Espaçamento** entre linhas de 1,5 ou duplo
- [ ] **Parágrafos curtos** (máximo 5-6 linhas)
- [ ] **Evitar texto justificado** (dificulta leitura)

### ✅ Imagens

- [ ] **Texto alternativo** (descrição) em todas as imagens
- [ ] **Legendas** explicativas
- [ ] **Contraste** adequado nas imagens
- [ ] **Não usar imagens com texto** (use texto mesmo)

### ✅ Tabelas

- [ ] **Cabeçalho** em todas as colunas
- [ ] **Linhas alternadas** (zebrado) para facilitar leitura
- [ ] **Título** descritivo da tabela
- [ ] **Não mesclar células** (dificulta leitores de tela)

### ✅ Links

- [ ] **Texto descritivo** (não use "clique aqui")
- [ ] **URLs curtas** ou usando encurtador
- [ ] **Verificar** se todos os links funcionam

---

## Checklist para Apresentações (Slides)

### ✅ Layout

- [ ] **Máximo 6 linhas** por slide
- [ ] **Máximo 6 palavras** por linha
- [ ] **Fonte mínima 24pt** (título 44pt+)
- [ ] **Contraste alto** entre texto e fundo
- [ ] **Fundo simples** (sem texturas)

### ✅ Conteúdo

- [ ] **Uma ideia** por slide
- [ ] **Títulos claros** e descritivos
- [ ] **Imagens grandes** e de qualidade
- [ ] **Não sobrecarregar** com informações

### ✅ Gráficos e Cores

- [ ] **Não usar cores** como única forma de transmitir informação
- [ ] **Padrões ou texturas** em gráficos (além das cores)
- [ ] **Legenda clara** para todos os elementos
- [ ] **Cores contrastantes** para daltonismo

---

## Checklist para Planilhas

### ✅ Estrutura

- [ ] **Cabeçalhos claros** em todas as colunas
- [ ] **Nome da aba** descritivo (não "Plan1")
- [ ] **Células mescladas** evitadas
- [ ] **Título** do documento na primeira linha

### ✅ Dados

- [ ] **Formatação consistente**
- [ ] **Células vazias** explicadas (N/A, -, etc.)
- [ ] **Validação** de dados quando possível
- [ ] **Proteção** de fórmulas importantes

## Recursos de Acessibilidade do Windows

### Ativar Recursos:

#### 🖼️ Lupa (para baixa visão)

```
Atalho: Windows + +

Uso:
- Windows + + → Ampliar
- Windows + - → Reduzir
- Windows + Esc → Fechar
```

#### 🗣️ Narrador (leitor de tela)

```
Atalho: Ctrl + Windows + Enter

Uso:
- Tab → Navegar entre elementos
- Enter → Selecionar
- Setas → Ler texto
- Caps Lock + Esc → Fechar
```

#### 🎨 Alto Contraste

```
Atalho: Alt Esquerdo + Shift Esquerdo + Print Screen

Ou:
Configurações → Acessibilidade → Alto contraste
```

#### ⌨️ Teclas de Aderência

```
Ativação: Pressionar Shift 5 vezes

Uso:
- Permite pressionar teclas uma de cada vez
- Útil para dificuldades motoras
```

#### 🖱️ Teclado Virtual

```
Caminho:
Configurações → Acessibilidade → Teclado → Teclado virtual

Ou:
Buscar "Teclado virtual" no menu Iniciar
```

## Como Criar Texto Alternativo

### Para Imagens:

**Descreva:**

1. **O que está na imagem** (conteúdo)
2. **Contexto** (por que está ali)
3. **Ação** (se houver pessoas fazendo algo)

**Exemplo ruim:**

```
❌ "Imagem"
❌ "Foto"
❌ "Gráfico"
```

**Exemplo bom:**

```
✅ "Gráfico de barras mostrando crescimento de vendas
    nos últimos 6 meses, com aumento de 25% em março"

✅ "Foto de reunião de equipe em sala de conferência,
    5 pessoas sentadas ao redor de mesa com laptops"
```

### No Word:

1. Clique com direito na imagem
2. Selecione "Ver texto alternativo"
3. Escreva a descrição
4. Feche

### No Google Docs:

1. Clique com direito na imagem
2. Selecione "Texto alternativo"
3. Escreva a descrição
4. Clique em "Confirmar"

## Teste de Acessibilidade

### Teste 1: Leitor de Tela

1. Ative o Narrador (Ctrl + Windows + Enter)
2. Tente navegar no documento só com o teclado
3. O leitor descreve as imagens?
4. Consegue entender a estrutura?

### Teste 2: Contraste

1. Imprima em preto e branco
2. Consegue ler tudo claramente?
3. Os gráficos ficam claros sem cores?

### Teste 3: Zoom

1. Aumente o zoom para 200%
2. O texto continua legível?
3. Nada sai do lugar ou fica cortado?

## Erros Comuns (Evite!)

### ❌ Erros de Estrutura:

- Títulos apenas com formatação (negrito, fonte maior)
- Listas criadas manualmente (traços, números digitados)
- Espaços para criar alinhamento

### ❌ Erros de Cor:

- Vermelho para erros (sem outro indicador)
- Verde para acertos (sem outro indicador)
- Cores pastéis em fundos
- Texto cinza claro em fundo branco

### ❌ Erros de Imagem:

- Imagens sem descrição
- Imagens com texto importante
- Gráficos com apenas cores
- Imagens muito pequenas ou borradas

### ❌ Erros de Texto:

- Fonte muito pequena (< 12pt)
- Texto justificado
- Parágrafos muito longos
- Abreviações sem explicação

- WebAIM Contrast Checker (webaim.org)
- Contrast Checker do Google

### Simular Daltonismo:

- Color Oracle (colororacle.org)
- Filtros do Windows

### Testar com Leitor de Tela:

- NVDA (gratuito): nvaccess.org
- Narrador do Windows (nativo)
